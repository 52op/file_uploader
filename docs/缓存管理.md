# 🗂️ Go 缓存管理指南

本文档介绍如何查看、自定义和管理 Go 的缓存目录。

## 📍 查看当前缓存位置

### 基本命令
```bash
# 查看模块缓存目录
go env GOMODCACHE

# 查看构建缓存目录
go env GOCACHE

# 查看所有环境变量
go env
```

### 默认位置
| 操作系统 | 模块缓存 | 构建缓存 |
|----------|----------|----------|
| Windows | `%USERPROFILE%\go\pkg\mod` | `%LocalAppData%\go-build` |
| Linux | `$HOME/go/pkg/mod` | `$HOME/.cache/go-build` |
| macOS | `$HOME/go/pkg/mod` | `$HOME/Library/Caches/go-build` |

## 🔧 自定义缓存路径

### Windows 设置
```powershell
# 临时设置（当前会话）
$env:GOMODCACHE = "D:\go-cache\mod"
$env:GOCACHE = "D:\go-cache\build"

# 永久设置（推荐）
go env -w GOMODCACHE=D:\go-cache\mod
go env -w GOCACHE=D:\go-cache\build

# 验证设置
go env GOMODCACHE
go env GOCACHE
```

### Linux/Mac 设置
```bash
# 临时设置
export GOMODCACHE="/custom/go/mod"
export GOCACHE="/custom/go/build"

# 永久设置（推荐）
go env -w GOMODCACHE=/custom/go/mod
go env -w GOCACHE=/custom/go/build

# 添加到 shell 配置文件（可选）
echo 'export GOMODCACHE="/custom/go/mod"' >> ~/.bashrc
echo 'export GOCACHE="/custom/go/build"' >> ~/.bashrc
```

## 📦 缓存迁移

### 查看缓存大小
```bash
# Windows
powershell "Get-ChildItem -Recurse $(go env GOMODCACHE) | Measure-Object -Property Length -Sum"

# Linux/Mac
du -sh $(go env GOMODCACHE)
du -sh $(go env GOCACHE)
```

### 迁移步骤

#### 1. 备份当前缓存位置
```bash
# 记录当前位置
echo "当前模块缓存: $(go env GOMODCACHE)"
echo "当前构建缓存: $(go env GOCACHE)"
```

#### 2. 创建新目录
```bash
# Windows
New-Item -ItemType Directory -Force -Path "D:\go-cache\mod"
New-Item -ItemType Directory -Force -Path "D:\go-cache\build"

# Linux/Mac
mkdir -p /custom/go/mod
mkdir -p /custom/go/build
```

#### 3. 复制缓存文件
```bash
# Windows
Copy-Item -Recurse "$(go env GOMODCACHE)\*" "D:\go-cache\mod\"
Copy-Item -Recurse "$(go env GOCACHE)\*" "D:\go-cache\build\"

# Linux/Mac
cp -r "$(go env GOMODCACHE)"/* /custom/go/mod/
cp -r "$(go env GOCACHE)"/* /custom/go/build/
```

#### 4. 更新环境变量
```bash
# 设置新路径
go env -w GOMODCACHE=/custom/go/mod
go env -w GOCACHE=/custom/go/build

# 验证设置
go env GOMODCACHE
go env GOCACHE
```

#### 5. 测试新缓存
```bash
# 测试构建（应该使用新缓存）
go build .

# 验证缓存工作正常
go mod download
```

## 🧹 缓存管理

### 查看缓存信息
```bash
# 查看缓存的模块
go list -m all

# 查看特定模块版本
go list -m -versions github.com/gin-gonic/gin

# 查看缓存统计
go clean -modcache -n
go clean -cache -n
```

### 清理缓存
```bash
# 清理模块缓存（谨慎使用）
go clean -modcache

# 清理构建缓存
go clean -cache

# 清理所有缓存
go clean -modcache -cache

# 清理当前项目的构建缓存
go clean
```

### 预热缓存
```bash
# 下载项目依赖到缓存
go mod download

# 下载所有版本的依赖
go mod download all

# 验证缓存完整性
go mod verify
```

## 💡 最佳实践

### 1. 缓存位置选择
- **SSD 存储**：将缓存放在 SSD 上提高构建速度
- **足够空间**：模块缓存可能占用几GB空间
- **备份考虑**：缓存可以重新下载，通常不需要备份

### 2. 团队协作
```bash
# 统一团队的模块代理
go env -w GOPROXY=https://goproxy.cn,direct

# 设置私有模块
go env -w GOPRIVATE=github.com/yourcompany/*

# 禁用校验和数据库（如果需要）
go env -w GOSUMDB=off
```

### 3. CI/CD 优化
```bash
# 在 CI 中缓存模块目录
# GitHub Actions 示例
- uses: actions/cache@v3
  with:
    path: |
      ~/.cache/go-build
      ~/go/pkg/mod
    key: ${{ runner.os }}-go-${{ hashFiles('**/go.sum') }}
```

### 4. 磁盘空间管理
```bash
# 定期清理旧的构建缓存
go clean -cache

# 只保留最近使用的模块
go clean -modcache
go mod download  # 重新下载当前项目需要的模块
```

## 🔍 故障排除

### 缓存损坏
```bash
# 清理并重新下载
go clean -modcache
go mod download
go mod verify
```

### 权限问题
```bash
# Linux/Mac 修复权限
sudo chown -R $USER:$USER $(go env GOMODCACHE)
sudo chown -R $USER:$USER $(go env GOCACHE)
```

### 网络问题
```bash
# 使用国内代理
go env -w GOPROXY=https://goproxy.cn,direct

# 或使用其他代理
go env -w GOPROXY=https://proxy.golang.org,direct
```

## 📊 缓存监控

### 查看缓存使用情况
```bash
# Windows
powershell "Get-ChildItem -Recurse $(go env GOMODCACHE) | Group-Object Extension | Sort-Object Count -Descending"

# Linux/Mac
find $(go env GOMODCACHE) -type f | wc -l
find $(go env GOCACHE) -type f | wc -l
```

### 定期维护脚本
```bash
#!/bin/bash
# go-cache-maintenance.sh

echo "Go 缓存维护脚本"
echo "==============="

echo "模块缓存位置: $(go env GOMODCACHE)"
echo "构建缓存位置: $(go env GOCACHE)"

echo "模块缓存大小: $(du -sh $(go env GOMODCACHE) | cut -f1)"
echo "构建缓存大小: $(du -sh $(go env GOCACHE) | cut -f1)"

# 清理超过30天的构建缓存
find $(go env GOCACHE) -type f -mtime +30 -delete

echo "缓存维护完成"
```

---

💡 **建议**：将缓存目录设置在 SSD 上，定期清理构建缓存，但保留模块缓存以提高下载速度。
