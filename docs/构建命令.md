# 🔧 Go 项目构建命令

本文档包含 Go 项目构建和依赖管理的常用命令。

## 🎯 Go Modules vs Vendor 选择

### 🚀 推荐：使用标准 Go Modules
- **简单高效**：无需维护额外的 vendor 目录
- **空间节省**：多项目共享全局缓存，避免重复存储
- **自动管理**：Go 工具链自动处理依赖下载和缓存
- **版本锁定**：go.sum 确保依赖完整性和可重现构建

### 📁 标准目录结构
```
file_uploader/
├── go.mod              # 模块定义和依赖声明
├── go.sum              # 依赖校验和版本锁定
├── main.go
└── ...
```

### ⚠️ Vendor 的局限性
- **存储重复**：vendor/ + 全局缓存双重占用空间
- **管理复杂**：需要手动维护 vendor 目录
- **收益有限**：现代开发环境网络稳定，离线构建需求较少

## 📦 依赖管理

### 初始化项目
```bash
# 初始化 Go 模块
go mod init file_uploader

# 整理依赖（清理未使用的依赖）
go mod tidy
```

### 添加和更新依赖
```bash
# 添加新依赖
go get github.com/gin-gonic/gin

# 更新所有依赖到最新版本
go get -u ./...

# 更新特定包
go get -u github.com/gin-gonic/gin

# 更新到特定版本
go get github.com/gin-gonic/gin@v1.9.1

# 整理依赖
go mod tidy
```

### 移除依赖
```bash
# 从代码中删除 import 后，运行
go mod tidy

# 或手动移除
go mod edit -droprequire github.com/some/package
```

## 🔨 构建命令

### 基本构建
```bash
# 标准构建
go build -o file_uploader.exe .

# Linux/Mac 构建
go build -o file_uploader .
```

### 跨平台构建
```bash
# Windows 64位
GOOS=windows GOARCH=amd64 go build -o file_uploader_windows.exe .

# Linux 64位
GOOS=linux GOARCH=amd64 go build -o file_uploader_linux .

# macOS 64位
GOOS=darwin GOARCH=amd64 go build -o file_uploader_macos .

# ARM64 (如 Apple M1)
GOOS=darwin GOARCH=arm64 go build -o file_uploader_macos_arm64 .
```

### 优化构建
```bash
# 去除调试信息，减小文件大小
go build -ldflags="-s -w" -o file_uploader.exe .

# 添加版本信息
go build -ldflags="-X main.version=1.0.0" -o file_uploader.exe .

# 静态链接（适用于 Linux）
CGO_ENABLED=0 go build -ldflags="-s -w" -o file_uploader .
```

## 🧪 测试命令

### 运行测试
```bash
# 运行所有测试
go test ./...

# 运行特定包的测试
go test ./handlers

# 运行测试并显示覆盖率
go test -cover ./...

# 详细测试输出
go test -v ./...
```

### 基准测试
```bash
# 运行基准测试
go test -bench=. ./...

# 运行特定基准测试
go test -bench=BenchmarkUpload ./handlers

# 基准测试 + 内存分析
go test -bench=. -benchmem ./...
```

## 🔍 依赖管理命令

### 查看依赖
```bash
# 查看当前依赖
go list -m all

# 查看依赖树
go mod graph

# 查看特定模块信息
go list -m -versions github.com/gin-gonic/gin

# 查看为什么需要某个依赖
go mod why github.com/some/package
```

### 验证依赖
```bash
# 验证依赖完整性
go mod verify

# 检查依赖
go list -deps ./...

# 下载依赖到本地缓存
go mod download
```

### 清理依赖
```bash
# 清理未使用的依赖
go mod tidy

# 清理模块缓存（谨慎使用）
go clean -modcache

# 重新下载所有依赖
go mod download
```

## 🚀 部署相关命令

### 打包发布
```bash
# 构建发布版本
go build -ldflags="-s -w" -o file_uploader.exe .

# 创建压缩包（Windows）
powershell Compress-Archive -Path file_uploader.exe,config,static,templates -DestinationPath file_uploader_release.zip

# 创建压缩包（Linux/Mac）
tar -czf file_uploader_release.tar.gz file_uploader config/ static/ templates/
```

### Docker 构建
```bash
# 使用 vendor 构建 Docker 镜像
docker build -t file_uploader .

# 多阶段构建（推荐）
docker build --target production -t file_uploader:latest .
```

## ⚠️ 注意事项

### 1. 依赖管理
- **版本锁定**：go.sum 文件确保依赖完整性，应提交到版本控制
- **定期更新**：定期运行 `go get -u ./...` 更新依赖到最新版本
- **安全扫描**：使用 `go list -json -deps` 检查依赖漏洞

### 2. 构建环境
- **Go 版本**：确保所有开发者使用相同的 Go 版本
- **模块代理**：可以设置 `GOPROXY` 使用私有代理或镜像
- **CI/CD**：确保 CI 环境能访问依赖源或使用缓存

### 3. 性能优化
- **模块缓存**：`$GOPATH/pkg/mod` 缓存可以加速构建
- **并行下载**：Go 1.11+ 支持并行下载依赖
- **离线模式**：使用 `GOPROXY=off` 强制使用本地缓存

### 4. 常用环境变量
```bash
# 设置模块代理（中国用户推荐）
export GOPROXY=https://goproxy.cn,direct

# 设置私有模块（跳过代理）
export GOPRIVATE=github.com/yourcompany/*

# 禁用模块校验（不推荐）
export GOSUMDB=off
```
